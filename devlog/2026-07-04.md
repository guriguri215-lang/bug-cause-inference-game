# Devlog: 2026-07-04

## Summary

Implemented P1b Phase C real-diff artifacts and connected `execution_grounded` `inspect_recent_diff` to those artifacts.

## Added

- Phase C1 real-diff artifact package under `src/bug_cause_inference/p1b/artifacts/real_diff/`.
- Clean checkout baseline source, 25 per-variant unified patches, and a manifest with lookup/validation fields only.
- Real-diff generator/validator that copies the baseline tree, applies one variant patch, scans generated source for variant branch tokens, validates manifest IDs against `load_p1b_variants()`, and checks generated checkout imports.
- Phase C2 observation wiring for `execution_grounded` `inspect_recent_diff`.
- Observation payload fields for `diff_artifact_path`, `changed_files`, `changed_functions`, and `diff_excerpt`.
- Markdown report rendering for recent-diff artifacts.
- C3 regenerated examples under `examples/p1b/`:
  - `p1b_variants.json` and `.md`
  - `p1b_report_P1B-BUG-001.json` and `.md`
  - `p1b_evaluation_summary.json` and `.md`

## Scope Notes

- `metadata_synth` remains the frozen synthetic baseline.
- `execution_grounded` now reads real-diff artifacts for `inspect_recent_diff`; test and coverage observations still come from the checkout execution harness.
- Diff evidence alone is not treated as a reproduced bug, so clean benign diffs do not become false positives by construction.
- No policy, threshold, or score tuning was introduced.
- No generated full source trees are committed.
- No patch generation, automated repair, LLM agent battle, adversarial generation, formal minimax framing, or P1c work was added.

## CI Update

Added a minimal GitHub Actions workflow for pull requests and pushes to `main`. It uses Python 3.10, installs `requirements.txt` plus the editable package, runs pytest, and validates the P1b real-diff artifacts.
